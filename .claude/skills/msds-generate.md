# MSDS Generation Skill (七层Pipeline架构)

## 支持类型
- **纯净物**: 单组分化学品，通过CAS号或化学名查询
- **混合物**: 多组分配方，需提供各组分名称、CAS、浓度

## 核心架构
```
L1 数据源(db/) → L2 实体标准化 → L3 证据检索融合 → L4 分类引擎 → L5 SDS生成(MD) → L6 Claude审查 → L7 PDF生成 → L8 输出归档
```

### ⚠️ 关键检查点：L6 Claude审查

**MD生成后、PDF生成前，Claude必须逐条阅读生成的MD文件进行审查。审查通过后才生成PDF。**

审查流程：
1. `sds_pipeline_v2.py` 生成 `.md` 文件
2. **Claude 读取并审查 `.md` 文件**（见下方审查清单）
3. 审查通过 → 调用 `pdf_generator.py` 生成 `.pdf`
4. 审查不通过 → 修复问题后重新生成 `.md`，再回到步骤2

**禁止在未经Claude审查的情况下直接从MD生成PDF。**

## 快速入口（推荐）

### 纯净物
```bash
python core/sds_pipeline_v2.py --query "乙醇"
python core/sds_pipeline_v2.py --query "64-17-5" --name "乙醇" --output output/pure/MSDS_ethanol.md
```

### 混合物
```bash
python core/sds_pipeline_v2.py --mixture "乙醇:64-17-5:30,甲醇:67-56-1:30,丙酮:67-64-1:15,丙三醇:56-81-5:25"
```

一键完成七层全流程，包含自动审查和来源追溯。

## 核心原则
1. **纯净物**: 知识库优先，相关度<80%时先建立知识库
2. **混合物**: 组分数据计算，不依赖知识库
3. **不瞎编**: 估算值需注明"估算"，鼓励实验验证
4. **数据来源**: 优先PubChem/知识库，LLM推断补全
5. **来源追溯**: 每个字段标注来源和置信度

## Phase 3: 混合物GHS分类计算

### 3.0 分类决策树

```
混合物
  ↓
有混合物整体实验数据？
  ├─ 是 → 直接使用实验数据分类
  └─ 否 → 能否使用桥接原则？
           ├─ 是 → 使用桥接原则推断
           └─ 否 → 基于组分数据分类
                    ├─ 可加和危害 → 使用ATE加和公式
                    └─ 非加和危害 → 使用切割值/浓度限值
```

### 3.1 可加和 vs 非加和危害类别

**可加和危害（使用ATE加和公式）**:
- 急性毒性（经口/经皮/吸入）
- 皮肤腐蚀/刺激
- 严重眼损伤/眼刺激
- STOT-单次接触-cat3（呼吸道刺激/麻醉效应）
- 水生环境危害

**非加和危害（使用浓度限值/切割值）**:
- 皮肤/呼吸致敏
- 生殖细胞致突变
- 致癌性
- 生殖毒性
- STOT-cat1和cat2
- 吸入危害

### 3.2 ATE加和公式（急性毒性）

**标准公式**:
```
1/ATE_mix = Σ(Ci / ATEi)

ATE_mix: 混合物的急性毒性估计值 (mg/kg)
Ci: 成分i的浓度（质量分数，如30%=0.30）
ATEi: 成分i的急性毒性估计值 (mg/kg)
```

**未知毒性成分>10%时的修正公式**:
```
1/ATE_mix = Σ(Ci/ATEi) / (100% - C_unknown)
```

**ATE值获取方式**:
- 已知LD50/LC50 → 直接使用实验值
- 仅有GHS分类 → 使用换算表

**GHS急性毒性分类换算表（口服）**:

| 分类 | LD50范围 (mg/kg) | 换算ATE值 (mg/kg) |
|------|-----------------|------------------|
| Cat 1 | ≤5 | 0.5 |
| Cat 2 | >5-50 | 5 |
| Cat 3 | >50-300 | 100 |
| Cat 4 | >300-2000 | 500 |
| Cat 5 | >2000-5000 | 2500 |

**分类阈值对照**:

| ATE计算结果 (mg/kg) | 混合物分类 |
|--------------------|-----------|
| ≤5 | 急性毒性 Cat 1, H300 |
| >5-50 | 急性毒性 Cat 2, H300 |
| >50-300 | 急性毒性 Cat 3, H301 |
| >300-2000 | 急性毒性 Cat 4, H302 |
| >2000-5000 | 急性毒性 Cat 5, H303 |

### 3.3 浓度限值（非加和危害）

**皮肤腐蚀/刺激**:

| 组分分类 | 混合物分类触发浓度 |
|---------|------------------|
| 皮肤腐蚀 Cat 1A/1B/1C | ≥5% → Cat 1 |
| 皮肤腐蚀 Cat 1A/1B/1C | ≥1%但<5% → Cat 2 |
| 皮肤刺激 Cat 2 | ≥10% → Cat 2 |
| 皮肤刺激 Cat 2 | ≥1%但<10% → 考虑分类 |

**严重眼损伤**:

| 组分分类 | 混合物分类触发浓度 |
|---------|------------------|
| 严重眼损伤 Cat 1 | ≥3% → Cat 1 |
| 眼刺激 Cat 2A | ≥10% → Cat 2A |
| 眼刺激 Cat 2A | ≥1%但<10% → 考虑分类 |

### 3.4 桥接原则（Bridging Principles）

| 原则 | 含义 | 适用条件 |
|------|------|---------|
| 稀释 | 已测试混合物A被较低危害成分B稀释 | A+B=A（分类不变） |
| 批次 | 同条件下生产的不同批次 | A=B |
| 高浓度混合物 | A已在最高危害类别分类，B浓度更高 | B=A |
| 插值 | A和B有相同危害成分和分类，C介于两者之间 | C=A=B |
| 实质相似 | A+B和C+B，A与C相似 | A+B=C+B |
| 气雾剂 | 非气雾形式已分类 | 气雾=非气雾 |

### 3.5 未知毒性成分处理

- ≤10%: 使用标准ATE公式
- >10%: 使用修正ATE公式
- ≥1%: 需在SDS和标签中注明"x%的混合物由未知急性毒性的成分组成"

### 3.6 计算示例

**示例: 乙醇30%(LD50=7060) + 甲醇30%(LD50=5628) + 丙酮15%(LD50=5800) + 丙三醇25%(LD50=12600)**

```
1/ATE = (0.30/7060) + (0.30/5628) + (0.15/5800) + (0.25/12600)
      = 0.0000425 + 0.0000533 + 0.0000259 + 0.0000198
      = 0.0001415

ATE = 7067 mg/kg → 急性口服毒性 Cat 5 (>2000-5000)，但实际值>5000，归为不分类
```

**易燃性判断**: 乙醇(Cat2)+甲醇(Cat2)+丙酮(Cat2)总浓度=75%≥10% → 混合物为易燃液体
取最低闪点组分(丙酮-20°C)为混合物闪点参考 → 易燃液体 Cat 2

### 混合物MSDS特殊内容

#### 第三部分：成分/组成信息（混合物必须用表格）

| 成分名称 | CAS号 | 浓度(%) | 危险分类 | SCL/M-factors |
|----------|-------|---------|----------|---------------|
| 乙醇 | 64-17-5 | 30 | 易燃液体-cat2, 眼刺激-cat2A | - |
| 甲醇 | 67-56-1 | 30 | 易燃液体-cat2, 急性毒性-cat3 | 经口ATE=100 mg/kg |

| 丙三醇 | 56-81-5 | 25 | 无重大危险 | - |

#### 第十一部分：毒理学信息（混合物需特别说明）

```markdown
**急性毒性估算（基于加权计算）**:
- 经口LD50（估算）: 计算值 mg/kg体重
⚠️ **重要说明**: 上述毒理学数据为基于组分的估算值，实际混合物毒性可能因组分相互作用而不同。
```

2. 遵循ADR/IMDG优先规则：
   - 急性毒性优先
   - 然后是腐蚀性
   - 然后是易燃性

# MSDS 16部分通用要求

## 第一部分：化学品及企业标识

**纯净物**:
- 化学品名称（中英文）、CAS号、分子式、分子量、UN编号、IUPAC名称

**混合物**:
- 化学品名称（配方名称）、配方编号、各组分列表、CAS（混合物无统一CAS）、UN编号

## 第二部分：危险性概述

**紧急概述**: 100-150字，描述主要危险、健康危害、环境危害

**GHS危险性类别**: 具体类别列表
**象形图**: 火焰/腐蚀/感叹号等
**信号词**: 危险/警告

**危险性说明（H码）**:
```
H225 极易燃液体和蒸气
H301 吞咽有毒
H314 造成严重皮肤灼伤和眼睛损伤
...
```

**防范说明（P码）**:
```
预防: P210...；P260...；P280...
响应: P302+P352...；P305+P351+P338...
储存: P403+P233...；P405...
废弃: P501...
```

## 第三部分：成分/组成信息

**纯净物**: 化学名称、CAS、纯度、化学类别

**混合物**: 必须用表格列出所有组分

## 第四部分：急救措施

```markdown
**吸入**: [40-60字]
**皮肤接触**: [40-60字]
**眼睛接触**: [40-60字]
**食入**: [40-60字]
**救援人员防护**: [具体装备]
**对医生提示**: [无特效解毒剂+对症治疗]
```

## 第五部分：消防措施

```markdown
**危险特性**: [50-80字]
**适用灭火剂**: [40-60字]
**禁止使用的灭火剂**: [30-50字]
**消防建议**: [60-80字]
```

## 第六部分：泄漏应急处理

```markdown
**人员防护**: [60-80字]
**应急响应**: [60-80字]
**环境保护**: [50-70字]
**泄漏隔离**: [60-80字]
**清理方法**: [60-80字]
```

## 第七部分：操作处置与储存

```markdown
**操作注意事项**: [80-100字]
**储存条件**: [80-100字]
**禁配物**: [60-80字]
```

## 第八部分：接触控制/个体防护

- 职业接触限值表格
- 监测方法、工程控制
- 呼吸/眼睛/皮肤防护、卫生措施

## 第九部分：理化特性

表格形式：外观、熔点、沸点、闪点、密度、溶解度等

**混合物估算值标注**: 如"约15-20°C（估算）"

## 第十部分：稳定性和反应性

- 稳定性、应避免条件、禁配物
- hazardous分解产物、聚合危害

## 第十一部分：毒理学信息

**纯净物**: 直接引用数据

**混合物**: 估算值+ATE加和公式计算+说明

## 第十二部分：生态学信息

- 生态毒性、持久性、生物富集潜力、土壤迁移性

## 第十三部分：废弃处置

- 处置方法、容器处理、注意事项

## 第十四部分：运输信息

| 项目 | 内容 |
|------|------|
| UN编号 | - |
| 正式运输名称 | - |
| 危险类别 | - |
| 包装类别 | - |
| 海洋污染物 | - |

## 第十五部分：法规信息

- 中国法规列表
- 国际/地区法规列表

## 第十六部分：其他信息

- SDS编号、版本、修订日期
- 参考文献、培训要求、免责说明

# L6 Claude审查清单

生成MD文件后，Claude必须逐条检查以下项目。**所有HIGH项必须通过，MEDIUM项需标注但可放行。**

## HIGH — 必须通过（任一不通过则退回修复）

| # | 检查项 | 说明 |
|---|--------|------|
| H1 | 模板路由正确性 | 化学品类型与模板匹配（如有机化学品不得使用酸/碱腐蚀模板） |
| H2 | GHS分类一致性 | S2分类列表 vs S11毒理学 vs S3组分分类 三处必须一致 |
| H3 | 运输分类正确性 | 易燃/有毒/腐蚀性化学品必须有正确的UN编号和危险类别，不得显示"不受限制" |
| H4 | S9易燃性字段 | 易燃化学品不得出现"不适用"，应填写实际分类 |
| H5 | H码/P码正确性 | H码与GHS分类严格对应，P码包含分类专项（如易燃需P210/P240） |
| H6 | 混合物化学兼容性 | 各组分之间不得存在不相容组合（如强酸+强碱、强氧化剂+强还原剂） |
| H7 | 信号词正确性 | Cat1/Cat2 → "危险"；Cat3-Cat5 → "警告"；无分类 → 无信号词 |
| H8 | 象形图正确性 | 与GHS分类严格对应（如腐蚀→GHS05，易燃→GHS02） |

## MEDIUM — 需检查但可标注放行

| # | 检查项 | 说明 |
|---|--------|------|
| M1 | S2紧急概述含外观描述 | 应包含物理状态、颜色、气味 |
| M2 | S9混合物数据来源标注 | 混合理化数据应注明"最低组分值"/"加权平均"等 |
| M3 | S11混合物双层结构 | 混合物需区分"产品整体评估"和"各组分详细信息" |
| M4 | S14按运输方式分段 | 应有公路铁路/海运IMDG/空运IATA三段 |
| M5 | 数据完整性 | 无过多"[待确认]"字段（≤3个为正常） |
| M6 | 重复/冗余文本 | 同一字段不得出现重复分类描述（如"类别2, 类别2"） |
| M7 | 混合物S9范围值 | 沸点/熔点等范围值应合理（不得出现如"-108~80°C"这样过宽范围） |

## 审查操作流程

```
1. Read 生成的 .md 文件
2. 逐项核对上述清单
3. 发现HIGH问题 → 修复代码/数据 → 重新生成 .md → 重新审查
4. 发现MEDIUM问题 → 标注但不阻塞，记录待后续优化
5. 全部HIGH通过 → 生成PDF
6. 输出审查结果摘要
```

# 保存与审查

## 保存路径

每种化学品单独文件夹，包含 MD + PDF + JSON 三件套。

纯净物：
```
./Bycrt/output/pure/{化学品名}/
  ├── MSDS_{化学品名}.md      ← L5生成，L6审查
```

## AI手动编辑链路（JSON覆写）

生成后的MSDS可通过JSON编辑→重新生成MD/PDF的链路进行修改：

### 步骤1: 导出可编辑JSON（生成时自动导出）
```bash
# 生成时自动在子文件夹内创建.json文件
python core/sds_pipeline_v2.py --query "乙醇" --name "乙醇"
# 产出: output/pure/乙醇/MSDS_乙醇.json
```

### 步骤2: 编辑JSON
打开 `.json` 文件，修改需要调整的字段值。JSON结构按16部分组织，每个字段包含 `value` 和 `source`。

### 步骤3: 应用修改，重新生成MD/PDF
```bash
python core/sds_pipeline_v2.py --override "output/pure/乙醇/MSDS_乙醇.json"
```

### 审查+编辑模式
```bash
python core/sds_pipeline_v2.py --query "乙醇" --review-edit
# 自动审查高风险字段，输出问题列表和JSON路径
# 编辑JSON后用 --override 应用
```

**Claude使用场景**: 当Claude审查发现具体字段问题时，可直接编辑JSON文件，然后运行 `--override` 重新生成，无需修改代码。
