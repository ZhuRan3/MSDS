# MSDS Generation Skill (Skill+Claude+RAG架构)

## 核心架构
```
用户 → Claude Code → RAG检索 → 知识库没有则先建立 → Claude直接生成MSDS → Review审查
```

## 核心原则
1. **知识库优先**：RAG检索相关度<80%时，必须先建立知识库再生成
2. **不瞎编**：绝不以低相关度化学品数据作为参考生成MSDS
3. **数据来源**：优先PubChem API获取，API没有则LLM推断

## 工作流程

### Phase 1: RAG检索

执行Python脚本获取化学品数据：
```bash
python msds_rag_client.py --query "<CAS号或化学名>"
```

RAG脚本返回：
- `has_knowledge`: 知识库是否命中
- `retrieved_chemicals`: 相关化学品列表（最多3个）
- `primary_chemical`: 主匹配化学品完整数据
- `context`: 用于LLM的上下文字符串

### Phase 2: 知识库建设（当RAG未命中或相关度<80%时）

如果RAG检索结果的相关度<80%或找不到该化学品，**必须先建立知识库**：

```bash
# 添加化学品到知识库（使用 --add CAS --name "中文名"）
python kb_manager.py --add "<CAS号>" --name "<中文名>"
```

kb_manager.py会：
1. 从PubChem API获取基础数据
2. 使用LLM推断详细属性
3. 保存到知识库

添加完成后，**重新执行RAG检索**确认知识库命中。

### Phase 3: MSDS生成（16部分）

按照GB/T 16483-2008标准生成完整16部分，每部分都要详细：

#### 第一部分：化学品及企业标识
- 表格形式，包含：化学品名称（中英文）、CAS号、分子式、分子量、UN编号、IUPAC名称
- 企业信息：名称/地址/应急电话留空（企业填写）
- 推荐用途/限制用途

#### 第二部分：危险性概述（最重要！）
```markdown
**紧急概述**: [100-150字，详细描述主要危险、健康危害、环境危害]

**GHS危险性类别**:
- [具体类别列表]

**象形图**: [火焰/腐蚀/感叹号等]
**信号词**: [危险/警告]

**危险性说明（H码）**:
- H314 [对应描述]
- H318 [对应描述]
- H335 [对应描述]

**防范说明（P码）**:
**预防**: P210...；P260...；P280...
**响应**: P302+P352...；P305+P351+P338...；P312...；P363...
**储存**: P403+P233...；P405...
**废弃**: P501...
```

#### 第三部分：成分/组成信息
- 物质类型：物质/混合物
- 化学名称、CAS、纯度、化学类别

#### 第四部分：急救措施（重要！）
```markdown
**吸入**: [40-60字：症状+撤离+供氧+人工呼吸+就医]
**皮肤接触**: [40-60字：脱衣+冲洗时间15-30分钟+就医+后续处理]
**眼睛接触**: [40-60字：冲洗时间+取隐形眼镜+就医]
**食入**: [40-60字：禁止催吐原因+漱口+禁止诱导呕吐+就医]
**救援人员防护**: [具体防护装备清单]
**对医生提示**: [无特效解毒剂+对症治疗+观察指标]
```

#### 第五部分：消防措施（重要！）
```markdown
**危险特性**: [50-80字：燃烧性+蒸气密度>空气+爆炸极限+腐蚀性+燃烧产物毒性]
**适用灭火剂**: [40-60字：具体类型+使用理由]
**禁止使用的灭火剂**: [30-50字：原因]
**消防建议**: [60-80字：呼吸器+防护服+上风方向+冷却容器+疏散]
```

#### 第六部分：泄漏应急处理
```markdown
**人员防护**: [60-80字：呼吸防护+防化服+手套+护目镜+冲洗]
**应急响应**: [60-80字：隔离距离50米+下风向疏散200米+上风操作]
**环境保护**: [50-70字：防止进入排水沟/水体/土壤]
**泄漏隔离**: [60-80字：小量/大量不同处理方法]
**清理方法**: [60-80字：中和剂+吸附材料+废液处理]
**应急人员防护装备**: [40-60字：具体装备清单]
```

#### 第七部分：操作处置与储存
```markdown
**操作注意事项**: [80-100字：密闭操作+局部排风+自动化+培训+禁止吸烟饮食+防护装备]
**储存条件**: [80-100字：阴凉干燥+温度<30℃+远离热源+容器密封+分类储存]
**禁配物**: [60-80字：具体化学品名称列表]
```

#### 第八部分：接触控制/个体防护
- 职业接触限值表格
- 监测方法、工程控制
- 呼吸防护、眼睛防护、皮肤防护、卫生措施

#### 第九部分：理化特性
表格形式：外观、熔点、沸点、闪点、自燃温度、爆炸极限、蒸气压、密度、溶解度

#### 第十部分：稳定性和反应性
- 稳定性、应避免的条件、禁配物
- hazardous分解产物、聚合危害

#### 第十一部分：毒理学信息（重要！）
```markdown
**急性毒性**: LD50经口/经皮/吸入 + 实验动物
**皮肤腐蚀/刺激**: 具体表现 + GHS分类依据
**严重眼损伤/眼刺激**: 具体表现 + 视力风险
**呼吸/皮肤致敏**: 分类依据
**致癌性**: IARC/NTP/OSHA分类
**生殖毒性**: 动物实验结果 + 分类依据
**单次接触STOT**: 靶器官 + 症状 + 麻醉效应
**反复接触STOT**: 靶器官 + 长期后果
```

#### 第十二部分：生态学信息
- 生态毒性：LC50/EC50数值
- 持久性和降解性、生物富集潜力、土壤迁移性
- 其他不良效应

#### 第十三部分：废弃处置
- 处置方法、容器处理、注意事项

#### 第十四部分：运输信息
表格：UN编号、正式运输名称、危险类别、包装类别、海洋污染物

#### 第十五部分：法规信息
- 中国法规列表
- 国际/地区法规列表

#### 第十六部分：其他信息
- SDS编号、版本、修订日期、编制者
- 培训要求、参考文献、免责说明

### Phase 4: 保存输出

生成完成后：
1. 保存到 `./Bycrt/{化学品中文名}/MSDS_{化学品名}_detailed.md`
2. 同时保存一份到 `./Bycrt/MSDS_{CAS}_detailed.md`

### Phase 5: 触发审查

MSDS生成完成后，**必须触发审查流程**：

执行审查脚本：
```bash
python msds_reviewer.py --input "<生成的MSDS文件路径>"
```

审查内容：
- 完整性检查（16部分是否齐全）
- GHS分类正确性（H码/P码/象形图是否匹配）
- 数据一致性（CAS号、分子式、危险类别是否一致）
- 格式规范性（是否符合GB/T 16483-2008）
- 专业知识检查（急救措施、消防建议是否合理）

审查结果输出格式：
```
## 审查报告

### 完整性: PASS/FAIL
[缺失部分列表]

### GHS一致性: PASS/FAIL
[问题列表]

### 数据一致性: PASS/FAIL
[问题列表]

### 格式规范性: PASS/FAIL
[问题列表]

### 建议:
[改进建议]
```

## 示例调用流程

### 1. 用户输入
```
/msds-generate 108-95-2
```

### 2. Claude执行
```bash
# Step 1: RAG检索
python msds_rag_client.py --query "108-95-2"
```

### 3. 知识库已有该化学品（相关度100%）
→ 直接进入MSDS生成

### 4. 知识库没有该化学品（相关度<80%）
```bash
# Step 2: 建立知识库
python kb_manager.py --add "108-95-2" --name "苯酚"

# Step 3: 重新检索确认
python msds_rag_client.py --query "108-95-2"
```

### 5. Claude生成完整MSDS

### 6. 触发审查
```bash
python msds_reviewer.py --input "./Bycrt/phenol/MSDS_phenol_detailed.md"
```

## 重要原则

1. **知识库必须完整**：RAG相关度<80%时，必须先建立知识库
2. **内容详尽**：每部分都要充分描述，不省略
3. **专业术语**：使用准确的化学、安全术语
4. **数据准确**：数值来自RAG或合理推断，不编造
5. **符合GHS**：H码/P码/象形图正确对应
6. **中文输出**：所有内容简体中文
7. **必须审查**：生成后必须触发审查流程
8. **不瞎搞**：绝不以低相关度化学品数据作为参考
