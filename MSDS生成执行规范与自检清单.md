## 1. 文档说明

### 1.1 文档用途

本文档是"化安通"(HuaAnTong)项目——AI驱动的MSDS自动生成系统的**核心执行规范**。所有MSDS生成代码必须严格遵循本文档执行，生成后的文档也必须按照本文档进行自检验证。

### 1.2 适用范围

- **系统范围**: 化安通MSDS自动生成系统全部模块
- **文档范围**: 所有中文化学品安全技术说明书(SDS/MSDS)
- **法规范围**: 中华人民共和国境内生产、进口、经营、使用的化学品
- **化学品类型**: 纯物质、混合物、金属、溶液、制剂等

### 1.3 版本管理

| 版本 | 日期 | 修订内容 | 修订人 |
|------|------|----------|--------|
| V1.0.0 | 2026-04-19 | 初始版本 | 化安通项目组 |
| V2.0.0 | 2026-04-19 | 全面审计修订：补充法规更新、完善16部分字段、新增安全标签规范、新增出口适配、完善自检清单、修正不一致项 | 化安通项目组 |

### 1.4 法律效力声明

根据《中华人民共和国危险化学品安全法》(2025年12月27日公布，2026年5月1日施行)相关条款，MSDS必须符合国家标准。该法共10章127条，是中国**首部**危化品专门法律，从行政法规升级为国家法律。

**关键法律条款**：
- **第三十一条**：危险化学品生产企业、进口企业应当提供与其生产、进口的危险化学品相符的**中文化学品安全技术说明书(SDS)**，并在包装（包括外包装）上粘贴中文化学品安全标签
- **第三十二条**：危险化学品生产企业、经营企业在销售时，应当向购买方提供符合法律和国家标准要求的SDS，不得经营没有SDS的危险化学品
- **第八十二条（运输环节）**：托运危险化学品时，托运人应向承运人提交危险货物托运清单，说明种类、数量、危险特性及应急处置措施，并提供化学品安全技术说明书
- **第七章（危险化学品登记）**：强化登记管理，要求通过危险化学品登记综合服务系统完成登记
- **第九章（法律责任）**：处罚力度大幅加重，明确企业主体责任

**违规处罚**：

| 违规情形 | 处罚措施 | 法律依据 |
|----------|----------|----------|
| 未提供SDS或标签不合规 | 责令改正，处10万元以下罚款 | 安全法第九章 |
| 逾期未改正 | 处10-20万元罚款，直接责任人处2-5万元罚款 | 安全法第九章 |
| 经营无SDS/标签危化品 | 同上 | 安全法第九章 |
| 情节严重 | 责令停产停业整顿，吊销相关许可证件 | 安全法第九章 |
| 提供虚假SDS信息 | 从重处罚 | 安全法第九章 |

## 3. MSDS文档总体格式规范

### 3.1 文档标题规范

| 项目 | 规范要求 | 编码实现 |
|------|----------|----------|
| 主标题 | **化学品安全技术说明书** | 固定字符串 |
| 副标题 | 按照GB/T 16483、GB/T 17519编制 | 固定字符串 |
| 字体要求 | 主标题黑体/加粗，字号大于正文 | 输出时控制 |
| 位置 | 文档最顶部，居中或左对齐 | 格式化输出 |

### 3.2 声明行规范

| 声明类型 | 标准表述 | 使用场景 |
|----------|----------|----------|
| 标准声明 | 按照GB/T 16483、GB/T 17519编制 | 默认使用 |
| 详细声明 | 根据GB/T 16483-2008《化学品安全技术说明书 内容和项目顺序》和GB/T 17519-2013《化学品安全技术说明书编写指南》编制 | 正式版本 |

### 3.3 编号与日期规范

| 字段 | 格式要求 | 示例 | 编码规则 |
|------|----------|------|----------|
| SDS编号 | 企业自定义编码系统 | SDS-2026-001, AH-ACN-001 | 企业编码规则 |
| 版本号 | X.Y.Z格式 | 1.0.0, 8.0 | 主版本.次版本.修订 |
| 修订日期 | YYYY-MM-DD | 2026-04-19 | ISO 8601格式 |
| 最初编制日期 | YYYY-MM-DD | 2020-01-15 | ISO 8601格式 |

### 3.4 页码规范

| 格式类型 | 中文格式 | 英文格式 | 推荐 |
|----------|----------|----------|------|
| 格式A | 第X页 共Y页 | Page X of Y | ★★★★★ |
| 格式B | 页 X/Y | X/Y | ★★★☆☆ |

**页脚重复信息规范**:
- 每页底部必须重复：化学品名称（中文）+ SDS编号
- 格式示例：`丙酮 | SDS-2026-001`

### 3.6 单位与数值格式规范

| 物理量 | 标准单位 | 格式要求 | 示例 |
|--------|----------|----------|------|
| 温度 | ℃（摄氏度） | 数值+单位，不空格 | 56.5℃ |
| 压力 | kPa（千帕） | 数值+单位，注明条件 | 24 kPa（20℃） |
| 密度 | 无量纲（水=1） | 标明基准 | 0.79（水=1） |
| 质量/体积浓度 | mg/m³ | 数值+单位 | 300 mg/m³ |
| 百分比浓度 | %（质量分数） | 注明"质量分数" | ≥99.5%（质量分数） |
| 致死剂量 | mg/kg | 注明动物种类和暴露途径 | 5800 mg/kg（大鼠，经口） |
| 致死浓度 | mg/L | 注明暴露时间和动物 | 50 mg/L/4h（大鼠，蒸气） |
| pH值 | 无量纲 | 单独数值 | 7.0 |
| 分子量 | g/mol | 保留两位小数 | 58.08 |

### 3.7 通用表述规范

| 场景 | 标准表述 | 禁止表述 | 说明 |
|------|----------|----------|------|
| 无数据 | "无资料" | "无数据"、"N/A"、"未知" | GB/T 17519规定用语 |
| 不适用 | "不适用" | "-"、"无" | 需说明原因 |
| 无危害 | "无" 或 "该产品不具有此类危害" | "0"、"否" | 需明确说明 |
| 数据范围 | "50～100" | "50-100" | 范围号使用"～" |
| 温度范围 | "-20～60℃" | "-20℃-60℃" | 公共单位写在末尾 |

## 4. 16个部分详细规范

### 4.1 第一部分：化学品及企业标识

| 序号 | 字段名称 | 字段标识 | 数据类型 | 填写规则 | 验证规则 |
|------|----------|----------|----------|----------|----------|
| 1.1 | 化学品中文名 | chemical_name_cn | String | 与标签一致，优先使用IUPAC名 | 非空，长度≤100字符 |
| 1.2 | 化学品英文名 | chemical_name_en | String | 与标签一致，优先使用英文IUPAC名 | 非空，长度≤200字符 |
| 1.3 | 企业名称 | company_name | String | 完整企业全称 | 非空，长度≤200字符 |
| 1.4 | 企业地址 | company_address | String | 完整到街道门牌号 | 非空，包含省/市/区/街道/门牌 |
| 1.5 | 企业邮编 | company_zip | String | 6位数字 | 正则：^\d{6}$ |
| 1.6 | 企业电话 | company_phone | String | 带区号的完整号码 | 正则：包含区号+号码 |
| 1.7 | 企业邮箱 | company_email | String | 有效邮箱格式 | 正则：标准邮箱格式 |
| 1.8 | 24h应急电话 | emergency_phone | String | 中国境内24小时电话 | 非空，进口产品必须提供中国境内电话 |
| 1.9 | 产品推荐用途 | recommended_use | String | 主要用途描述 | 非空，简明扼要 |
| 1.10 | 产品限制用途 | restricted_use | String | 禁止或限制的用途 | 无则填"无" |

#### 常见错误

| 错误类型 | 错误示例 | 正确做法 |
|----------|----------|----------|
| 名称不一致 | 中文名用"丙酮"，英文名用"dimethyl ketone"而不用"acetone" | 英文名优先使用最常用名"acetone" |
| 地址不完整 | "天津市南开区" | 必须到街道门牌："天津市南开区卫津路92号" |
| 缺少24h电话 | 只提供工作时间电话 | 必须提供24小时应急电话 |
| 进口产品无中国电话 | 提供美国800电话 | 必须提供中国境内可拨打的电话 |
| 分子式格式错误 | "c3h6o" | "C3H6O"（元素符号大写） |

### 4.2 第二部分：危险性概述

#### 必填项清单

| 序号 | 字段名称 | 字段标识 | 数据类型 | 填写规则 | 验证规则 |
|------|----------|----------|----------|----------|----------|

| 2.4 | 信号词 | signal_word | String | "危险"或"警告" | 二选一 |
| 2.5 | 危险性说明 | h_statements | String[] | H语句代码+中文 | 对应GHS类别 |

## 6. 混合物特殊规则

### 6.1 混合物分类方法

| 方法 | 说明 | 适用场景 |
|------|------|----------|
| 试验法 | 对混合物本身进行试验 | 混合物作为整体有试验数据 |
| 架桥原则 | 使用类似混合物的数据推断 | 有类似混合物的数据 |
| 加和公式 | 根据组分浓度和分类计算 | 有所有组分的分类数据 |
| 截断值/浓度限值 | 根据组分浓度和限值判断 | 组分浓度≥限值 |

### 6.2 架桥原则

| 原则 | 规则 |
|------|------|
| 稀释原则 | 稀释后混合物的分类与原混合物相同（稀释剂危险性更低） |
| 批次间差异原则 | 一批已分类，其他批次可认为相同 |
| 浓度相近混合物原则 | 混合物B的组分与A相同，浓度相近时可认为分类相同 |
| 内插法原则 | 混合物B的组分浓度在A和C之间，可认为分类在A和C之间 |
| 实质上类似的混合物原则 | 用毒性更低的组分替换，新混合物分类与原混合物相同 |
| 气溶胶原则 | 装入气溶胶容器后分类与原混合物相同 |

### 6.3 加和公式

#### 6.3.1 急性毒性加和公式

```
100 / LDMix = Σ (Ci / LDi)
```

#### 6.3.2 皮肤腐蚀/刺激加和公式

| 条件 | 规则 |
|------|------|
| 类别1组分总和≥5% | 混合物为类别1 |
| 类别1组分总和<5%但类别2组分总和≥10% | 混合物为类别2 |

#### 6.3.3 严重眼损伤/眼刺激加和公式

| 条件 | 规则 |
|------|------|
| 类别1组分总和≥3% | 混合物为类别1 |
| 类别1组分总和<3%但类别2组分总和≥10% | 混合物为类别2 |

### 6.5 M因子（乘数因子）详细规范

M因子适用于**对水环境具有高毒性的物质**（LC50/EC50 < 1 mg/L），在混合物的加和法分类中使用。

#### 6.5.1 急性毒性M因子表

| L(E)C50值 (mg/L) | M因子 |
|------------------|-------|
| 0.1 < L(E)C50 ≤ 1.0 | 1 |
| 0.01 < L(E)C50 ≤ 0.1 | 10 |
| 0.001 < L(E)C50 ≤ 0.01 | 100 |
| 0.0001 < L(E)C50 ≤ 0.001 | 1,000 |
| 0.00001 < L(E)C50 ≤ 0.0001 | 10,000 |

#### 6.5.2 慢性毒性M因子表

| NOEC值 (mg/L) | M因子（不可快速降解） | M因子（可快速降解） |
|---------------|---------------------|-------------------|
| 0.01 < NOEC ≤ 0.1 | 1 | 1 |
| 0.001 < NOEC ≤ 0.01 | 10 | 1 |
| 0.0001 < NOEC ≤ 0.001 | 100 | 10 |
| 0.00001 < NOEC ≤ 0.0001 | 1,000 | 100 |

#### 6.5.3 含M因子的水生毒性加和法公式

**急性分类**：
```
Σ(急性1类浓度% × Mi) ≥ 25%     → 混合物为急性1类
Σ(急性2类浓度%) + Σ(急性1类浓度% × Mi × 10) ≥ 25%  → 混合物为急性2类
Σ(急性3类浓度%) + Σ(急性2类浓度%) + Σ(急性1类浓度% × Mi × 100) ≥ 25%  → 混合物为急性3类
```

**慢性分类**：
```
Σ(慢性1类浓度% × Mi) ≥ 25%     → 混合物为慢性1类
Σ(慢性2类浓度%) + Σ(慢性1类浓度% × Mi × 10) ≥ 25%  → 混合物为慢性2类
Σ(慢性3类浓度%) + Σ(慢性2类浓度%) + Σ(慢性1类浓度% × Mi × 100) ≥ 25%  → 混合物为慢性3类
```

```python
    results.extend(check_section_15(msds_document))
    results.extend(check_section_16(msds_document))
    
    # GHS分类检查
    results.extend(check_ghs_classification(msds_document))
    
    # 混合物检查（如适用）
    if is_mixture(msds_document):
        results.extend(check_mixture_rules(msds_document))
    
    return generate_report(results)
```
